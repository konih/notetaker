# Test media fixtures

Sample **WAV** and **MP4** files for developing and testing Notetaker: English/German **meeting speech** for the live recorder path, **presentation videos** for `transcribe-video` (audio extract + slide detection), and **local media for e2e/integration smoke tests** (no live Teams call or microphone — see [E2E tests](#e2e-tests-testse2e) below).

Files live under `tests/fixtures/`. Refresh everything with:

```bash
task install:video-prereqs   # ffmpeg + yt-dlp
task fixtures:fetch
```

Python helpers: `tests/fixtures/paths.py` (`MEETING_EN_WAV`, `PRESENTATION_EN_VIDEO`, etc.).

---

## E2E tests (`tests/e2e/`)

Smoke e2e exercises **CLI → container → SQLite** without a live Teams call, microphone, or cloud STT. Most modules use **mocks** and a temp database only. Video modules call **real ffmpeg** on a **synthetic presentation MP4** generated per test run via `scripts/generate_sample_video.py` (same output as `task fixtures:sample-video`; no committed file required).

```bash
task install:video-prereqs   # ffmpeg required for video e2e modules
task test:e2e                # uv run pytest tests/e2e -q
```

| E2e module | Media | Notes |
|------------|-------|--------|
| `test_cli_record_smoke.py` | None | Mocked recorder + STT |
| `test_cli_sessions_smoke.py` | None | Mocked |
| `test_cli_finalize_smoke.py` | None | Mocked WhisperX |
| `test_cli_transcribe_video_smoke.py` | Per-run synthetic MP4 | Real ffmpeg audio extract + mocked transcriber; skips if ffmpeg missing |
| `test_cli_cleanup_and_slides_smoke.py` | Per-run synthetic MP4 | Cleanup CLI + slide preview; real slide detection |

Helper: `tests/e2e/video_helpers.py` (`generate_sample_video`, `ffmpeg_available`).

**Committed fixtures** (`wav/`, `video/`, optional `sample_presentation.mp4` from `task fixtures:fetch`) support manual CLI checks and `@pytest.mark.integration` tests — current e2e modules do not import them. **Meeting WAVs** may back future e2e that exercise `record` against fixture audio instead of mocks.

Path constants for integration tests and manual runs: `tests/fixtures/paths.py` (`MEETING_EN_WAV`, `PRESENTATION_EN_VIDEO`, `fixtures_available`, etc.).

## Sample videos (presentations)

Used for `live-transcriber transcribe-video`, slide detection e2e, and video import integration tests.

| File | Duration | Content | Source |
|------|----------|---------|--------|
| `video/presentation_en_15s_360p.mp4` | 15 s | English tech talk with on-screen slides (Kafka / Strimzi migration) | YouTube [DZL-ExKPjnc](https://www.youtube.com/watch?v=DZL-ExKPjnc) — clipped from ~30 s in |
| `video/presentation_en_120s_360p.mp4` | **2 min** | Same talk, longer clip for chunk transcription + slide tuning | Same source, 120 s from ~30 s in |
| `video/presentation_de_15s_360p.mp4` | 15 s | German conference talk with slides | [36C3: Nutzung öffentlicher Klimadaten](https://media.ccc.de/v/36c3-10571) (CC BY 4.0), CDN stream clipped from ~120 s |
| `video/presentation_de_120s_360p.mp4` | **2 min** | Same German talk, longer clip | Same CCC source |
| `sample_presentation.mp4` | 45 s | Synthetic deck: three solid-color slides + sine tone audio | `scripts/generate_sample_video.py` (gitignored; regenerated by `task fixtures:fetch`) |
| `sample_presentation_120s.mp4` | **2 min** | Synthetic deck: eight 15 s slides (deterministic slide detection) | Same generator (gitignored) |

**Try locally:**

```bash
task fixtures:fetch

# Real English presentation clip
uv run live-transcriber transcribe-video \
  --source tests/fixtures/video/presentation_en_15s_360p.mp4 \
  --title "Fixture EN presentation"

# Real English presentation (2 min)
uv run live-transcriber transcribe-video \
  --source tests/fixtures/video/presentation_en_120s_360p.mp4 \
  --title "Fixture EN presentation (2 min)"

# Deterministic slide detection (3 slides)
uv run live-transcriber transcribe-video \
  --source tests/fixtures/sample_presentation.mp4 \
  --yes-slides

# Real 15 s clips: expect ~2–3 on-screen slide changes (not a fixed deck).
# Defaults cap min-interval on short clips; for every transition use:
uv run live-transcriber transcribe-video \
  --source tests/fixtures/video/presentation_en_15s_360p.mp4 \
  --preview-only \
  --min-interval 0 --threshold 0.05 --sample-interval 1
```

**E2e / CI:** video e2e modules generate a fresh synthetic MP4 per run (see [E2E tests](#e2e-tests-testse2e)). The integration test mocks the download seam and imports the committed `presentation_en_15s_360p.mp4` fixture (deterministic, no network); it runs in CI's `integration` job.

**Override English source:** set `FIXTURE_PRESENTATION_EN_URL` before `task fixtures:fetch`, or `SAMPLE_VIDEO_URL` for `task fixtures:download-sample` (cache only).

## Sample audio (meetings)

Used for STT smoke tests, audio level utilities, and any test that needs realistic **spoken** English or German at 16 kHz mono PCM (`AUDIO_SAMPLE_RATE` default).

| File | Duration | Content | Source |
|------|----------|---------|--------|
| `wav/meeting_en_10s_16k_mono.wav` | 10 s | Multi-speaker **English meeting** audio | [AMI Meeting Corpus](https://groups.inf.ed.ac.uk/ami/corpus/) — `ES2002a` headset mix (CC BY 4.0), clipped from 600 s |
| `wav/meeting_en_120s_16k_mono.wav` | **2 min** | Same AMI session — **four speakers**, longer clip for chunk STT / diarization | Same `ES2002a` Mix-Headset, 120 s from 600 s |
| `wav/meeting_de_10s_16k_mono.wav` | 10 s | **German spoken** audio | Same 36C3 talk as `presentation_de_*` (audio only, CC BY 4.0) |
| `wav/meeting_de_120s_16k_mono.wav` | **2 min** | German conference speech (single primary speaker) | Same 36C3 talk, 120 s from ~120 s |

**German meeting note:** Public **German multi-speaker meeting** corpora with small direct downloads are rare. The German WAV is a spoken German **conference excerpt**, not an AMI-style meeting room. Validate real DE meeting capture manually (e.g. Teams + `record`).

## Task commands

| Task | What it does |
|------|----------------|
| `task fixtures:fetch` | Download/stream sources, truncate, normalize; write `wav/` + `video/` + synthetic `sample_presentation.mp4` |
| `task fixtures:sample-video` | Regenerate synthetic 3-slide MP4 only |
| `task fixtures:download-sample` | Cache full English presentation MP4 under `tests/fixtures/.cache/presentation_en_source.mp4` |

## Environment variables

| Variable | Default | Effect |
|----------|---------|--------|
| `FIXTURE_MEETING_SECONDS` | `10` | Short meeting WAV length |
| `FIXTURE_MEETING_LONG_SECONDS` | `120` | Long multi-speaker meeting WAV length |
| `FIXTURE_PRESENTATION_SECONDS` | `15` | Short presentation MP4 length |
| `FIXTURE_PRESENTATION_LONG_SECONDS` | `120` | Long presentation MP4 length (2 min) |
| `FIXTURE_PRESENTATION_EN_OFFSET_SECONDS` | `30` | Start offset in English presentation (YouTube cache) |
| `FIXTURE_AMI_OFFSET_SECONDS` | `600` | Start offset in AMI meeting audio |
| `FIXTURE_CCC_OFFSET_SECONDS` | `120` | Start offset in CCC talk (DE video + DE WAV) |
| `FIXTURE_PRESENTATION_EN_URL` | `https://www.youtube.com/watch?v=DZL-ExKPjnc` | English presentation download |
| `FIXTURE_CCC_PRESENTATION_URL` | 36C3 CDN MP4 (see `scripts/fetch_test_fixtures.sh`) | German presentation stream |
| `FIXTURE_AMI_MEETING_URL` | AMI `ES2002a` Mix-Headset WAV | English meeting stream |
| `FIXTURE_SLIDE_SECONDS` | `15` | Seconds per slide in synthetic MP4 |
| `FORCE_FIXTURE_DOWNLOAD` | `0` | Set to `1` to re-download cached YouTube file |
